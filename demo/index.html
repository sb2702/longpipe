Longpipe demos

Pick one. Each is an isolated page so you can link/share/test individually.

  • Try Longpipe (website-style demo) Local mirror of the website's /demo page. Source toggle (sample video / webcam), all background modes (image, blur, color, video, none), preset picker. Imports from ~/pipeline so it tracks local SDK changes.
  • Audio denoise (mic → pipeline → playback) Audio-denoise subsystem (separate AudioWorklet). Mic + cam → EffectsPipeline with audio: {denoise} → playback. Model picker (dfn/dfnint8/rnnoise/auto), wet/dry toggle, β + GRU-leak sliders, per-hop stats. Weights served locally from /weights/. Use headphones.
  • Face-heatmap probe The face keypoint head in isolation: 5 heatmaps at base/4|8 painted over the source, windowed soft-argmax keypoints (toggle hard argmax to see the jitter it prevents), landmark-crop box preview, live jitter meter. Tier/backend/precision/source pickers.
  • Landmark probe (GPU-resident) The full landmark chain with zero per-frame readback: matting taps → heatmaps → soft-argmax box (compute) → 256² crop → LandmarkNet → 478 vertex-pulled dots. The stats line is an optional 16-byte readback every 30 frames.
  • Face touch-up probe The complete touch-up flow standalone: landmark chain → UV unwrap → freq-sep/bilateral smoothing → weight-mask composite. Compare toggle, strength/amount/detail/box-scale sliders.
  • Matting (static image) End-to-end matting on a still image. Pick backend (WebGPU/WebGL), precision (fp32/fp16), upscaler, and background mode.
  • Pipeline: looped video → Pipeline → video element Full SDK end-to-end. Looped video file feeds captureStream() as the input MediaStream; the output MediaStream drives a second video element. Real-world device test target.
  • transferControlToOffscreen + captureStream Pipeline plumbing test. Does a canvas's captureStream() keep emitting frames after its drawing control is transferred to a worker? Chromium yes, Firefox no.
  • Bitmap shuttle (worker → ImageBitmap → main canvas → captureStream) Universal-fallback path test. Worker renders to its own OffscreenCanvas, ships an ImageBitmap to main per frame, main draws it into a bitmaprenderer canvas that's captured.
  • MediaStreamTrackGenerator (worker writes VideoFrames) Best-path test for the output side. Main creates a generator and transfers its writable to the worker; worker writes VideoFrames; main wraps the generator in a MediaStream.
