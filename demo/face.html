Face-heatmap probe — multi-face decode

Runs the tier's production .bin end-to-end: TierModel matting pass → encoderTaps → FaceHeatmapNet → 5 keypoint heatmaps at base/4 (L-eye, R-eye, nose, L-mouth, R-mouth). Keypoints are decoded with the windowed soft-argmax centroid (the required decode — toggle it off to see hard-argmax cell snapping). Dashed squares are the crops the landmark stage would take.

multi-face (CPU prototype for the face_box.wgsl rewrite): local-max candidates per channel → every (L-eye, R-eye) pair becomes a hypothesis fixing center/scale/roll → nose + mouth corners predicted from the canonical template and matched within tol × interocular → scored (peak scores − geometric residual) → greedy NMS on consumed candidates. Off = the current single-face production path (global argmax + hull), for A/B. candidates shows stage 1 before grouping, so a missing face reads as "never detected" vs "grouping rejected it". jitter = per-keypoint position std over 30 frames for the top-scoring face, in canvas px.
